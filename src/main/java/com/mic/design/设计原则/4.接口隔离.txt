接口隔离原则

      1.实例接口 object Interface  Person person = new Person()
      2.类接口 Class Interface

      ------------------------------------------------------------
      1.接口尽量要小
      2.接口要高内聚，--提高接口，类，模块的处理能力，减少对外的交互
      3.接口的设计是有限度的

      ------------------------------------------------------------
      实践
      1.一个接口只服务于一个子模块或业务逻辑
      2.通过业务逻辑压缩接口中的public方法
      3.已经被污染了的接口，尽量去修改，若变更的风险较大，刚采用适配器模式去进行转化处理
